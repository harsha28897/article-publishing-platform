## AI Collaboration Summary

AI tools were used during this assignment mainly to:

* speed up development,
* help with Angular frontend implementation,
* validate backend architecture ideas,
* troubleshoot issues,
* generate documentation,
* create ERD and sequence diagrams.

Most of the backend architecture and API design decisions were based on prior experience with Java, Spring Boot, REST APIs, PostgreSQL, and Spring Security.

AI assistance was mainly useful in:

* Angular standalone component setup,
* frontend routing and guards,
* JWT integration between frontend and backend,
* debugging configuration issues,
* improving README documentation,
* organizing project structure,
* generating diagrams and documentation content.

Key areas where AI assistance was used:

* Spring Security and JWT authentication flow
* Angular frontend setup and troubleshooting
* API integration guidance
* Database ERD generation
* Sequence diagram generation
* Debugging frontend and backend issues
* Improving documentation quality

The project implementation was validated manually throughout development by:

* running the backend and frontend applications,
* testing APIs using Postman,
* verifying authentication flow,
* testing article creation and comments,
* verifying frontend routing and protected pages.

AI was used as a productivity and collaboration tool during the development process.
